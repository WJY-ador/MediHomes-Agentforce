# MediHomes Agentforce 데모 시나리오

## 개요

- **채널**: 웹챗 (Embedded Service)
- **AI**: Agentforce + Data Cloud 360 + Salesforce Knowledge
- **인증방식**: 이름 + 생년월일 → 단일 매칭 시 즉시 조회 / 동명이인 시 SMS OTP 인증

---

## 인증 흐름 공통 기준

```
이름 + 생년월일 입력
    ↓
Account 조회
    ├─ 1건 매칭 → 인증 성공 → 데이터 조회
    ├─ 동명이인 (2건 이상) → SMS OTP 발송 → 인증 성공 → 데이터 조회
    └─ 불일치 → 재시도 요청 (최대 3회) → 실패 시 고객센터 안내
```

> **Salesforce 표준 기능**: `Verify Customer Identity` Action (SMS OTP)
> Account의 `MobilePhone` 필드로 OTP 발송

## 시나리오 1: 초기 순응 케어 + 기기/소모품 FAQ (인증 불필요)

### 콘셉트
고객이 로그인 없이 웹챗에서 양압기 사용 관련 질문을 하면,
Agentforce가 Knowledge 기반으로 즉시 답변.
고객사 활용방안 1번(초기 순응 케어) + 3번(기기/소모품 정보) 커버.

### 예시 질문 및 예상 답변

**Q. "마스크 자국이 심해요"**
마스크 착용 후 얼굴에 자국이 생긴 경우에는 마스크 머리끈을 조금 더 느슨하게 하여 착용 부탁드립니다.
- 출처: 소모품

**Q. "바람이 새는 것 같아요"**
마스크 앞 Vent 부분을 통해 이산화탄소 배출 및 압력 조절을 위해 지속적으로 바람이 새어나오는 것은 정상입니다. 마스크 착용이 느슨하거나 구강호흡으로 인한 누출이 발생된 경우라면 마스크 착용을 견고하게 해주시고, 입벌림 방지 테이프 사용을 권장드립니다.
- 출처: 소모품/기기

**Q. "코막힘이 심한데 양압기 계속 써도 되나요?"**
비염이 있는 환자도 양압기 사용이 가능합니다. 담당의와 양압기 사용 관련하여 상담 부탁드립니다.
- 출처: 임상정보

**Q. "가습기 물이 너무 빨리 줄어들어요"**
주변 환경에 따라 양압기에서 가습량을 자동 조절하기 때문에, 주변이 건조할 경우 가습량이 많아집니다. 계절이나 실내 환경 변화에 따른 정상 작동입니다. 안내 후에도 불편하시다면 수동으로 가습 단계를 조절해드리겠습니다.
- 출처: 기기

**Q. "소모품 교체 주기가 어떻게 되나요?"**
소모품 교체 주기는 개인마다 다를 수 있습니다. 일반적으로 아래 주기를 권장드립니다.
- 호스 / 물통: 1년 이내 1회
- 필터: 2~3개월 1회 (레즈메드/피셔 3개월, 필립스 일반필터 6개월/미세필터 1개월)
- 마스크: 6개월~1년 이내 1회
- 출처: 소모품/세척 및 관리

**Q. "필터는 어떻게 세척하나요?"**
필터는 브랜드에 따라 교체 방법이 다릅니다.
- 레즈메드 / 피셔: 3개월 주기 무상교체이며 세척 없이 폐기 후 교체해주세요.
- 필립스(Dreamstation): 일반필터(진한파랑)는 2주 1회 세척 후 그늘 건조(6개월 교체), 미세필터(하늘색)는 세척 불가(1개월 교체)
- 출처: 세척 및 관리

**Q. "기기에서 소음이 나는데 고장인가요?"**
우선 기기를 평평한 곳에 두었는지, 모든 소모품 연결이 제대로 되어 있는지 확인 부탁드립니다. 확인 후에도 소음이 지속된다면 소음 발생 부위를 인지하신 후 고객센터로 연락 부탁드립니다.
- 출처: 기기 → 미해결 시 상담원 전환

**Q. "입이 건조해요"**
사용 후 입이 건조한 경우 수면 중 구강호흡 가능성이 있습니다. 턱끈이나 입벌림방지 테이프를 함께 사용해보시길 권장드리며, 지속적인 건조함을 느끼신다면 양압기 내 가습레벨을 높여 사용하실 수 있습니다.
- 출처: 임상정보

### Agent 흐름
1. 고객 질문 입력
2. Data Cloud → Knowledge Data Library 검색
3. FAQ 기반 답변 생성 및 전달
4. 필요 시 유튜브/이미지 링크 함께 제공
5. 기기이상 문의 시 → 자가진단 체크리스트 안내 후 미해결 시 상담원 전환 유도

### 활용 Knowledge 카테고리
| 카테고리 | 주요 내용 |
|----------|-----------|
| 세척 및 관리 | 마스크/호스/물통/필터 세척법, 건조법, 보관법 |
| 소모품 | 교체주기, 구매방법, 마스크 타입 선택 |
| 기기 | 브랜드별 사용법, 소음, 전원, 가습/온도 설정 |
| 임상정보 | 수면무호흡증, 코막힘, 구강건조, 압력 적응 |
| 보험임대 | 일반 보험 관련 공통 FAQ |
| 기타 | 해외출국, 매장위치 등 |

### Salesforce 구성 요소
| 구성요소 | 필요 여부 |
|----------|-----------|
| Knowledge (Lightning Knowledge) | ✅ 필수 |
| Data Cloud + Data Library | ✅ 필수 |
| Agentforce Action: Answer Questions with Knowledge | ✅ 필수 |
| Account / Prescription 조회 | ❌ 불필요 |

## 시나리오 2: 나의 정보 조회 (본인인증 후 Salesforce 데이터 답변)

### 콘셉트
고객이 본인의 계약/처방/기기 정보를 문의하면,
Agentforce가 이름 + 생년월일로 본인인증 후 Salesforce 데이터를 기반으로 맞춤 답변.
동명이인이면 SMS OTP로 추가 인증.
고객사 활용방안 1번(마스크 주문 확인) + 2번(나의 정보 조회) 커버.

### 예시 질문 및 예상 답변

**Q. "제 처방전 만료일이 언제인가요?"**
고객님의 처방만료일은 **[End_Date__c]** 입니다. 만료일에 맞춰 처방전을 갱신하여 제출해주시면 계약기간이 자동 연장됩니다.
- 데이터: `Prescription__c.End_Date__c` / `Manual_expiredate__c`

**Q. "이번 달 순응 시간이 얼마나 됐나요?"**
현재 하루 평균 사용시간은 **[Average_usage_time__c]** 입니다. 순응 조건은 월 평균 2시간 이상(3개월 기준 총 180시간)으로, 현재까지 **[usage_days__c]일** 달성하셨습니다.
- 데이터: `Prescription__c.Average_usage_time__c`, `usage_days__c`, `Compilance_startdate__c`, `Compilance_enddate__c`

**Q. "마스크 보험 구매 아직 가능한가요?"**
양압기 마스크 보험 혜택은 해당 연도(1/1~12/31) 내 1회 구매 가능합니다. [올해 구매 이력 확인] 후 가능/불가 안내드립니다.
- 구매 가능한 경우: "동일한 마스크([현재 사용 마스크 모델])로 주문하시겠어요?"
- 다른 마스크 원하시는 경우: 고객센터로 연결
- 데이터: `Order/OrderItem` (해당 연도 마스크 구매 이력)

**Q. "지금 쓰는 기기/마스크가 어떤 건가요?"**
고객님께서 현재 사용 중인 기기는 **[브랜드/모델명]**, 마스크는 **[마스크 모델명]** 입니다.
- 데이터: `Asset` (기기 브랜드, 모델명, 시리얼)

**Q. "마스크 자국이 심한데 제 마스크가 뭔지 알고 싶어요"**
현재 사용 중인 마스크는 **[마스크 모델명]** 입니다. 마스크 착용 후 자국이 생기는 경우 머리끈을 조금 더 느슨하게 조절해보시길 권장드립니다.
- 데이터: `Asset` + Knowledge(소모품 FAQ)

### Agent 흐름
1. 고객 질문 입력 (개인정보 관련 감지)
2. 이름 + 생년월일 입력 요청
3. Account 조회
   - 1건 매칭 → 인증 성공
   - 동명이인 → `Verify Customer Identity` Action → SMS OTP 발송 → 인증
   - 불일치 3회 → 고객센터 안내
4. 인증 성공 → Salesforce 데이터 조회 및 맞춤 답변
5. 마스크 보험 구매 가능 → 동일 마스크 재주문 여부 확인 → 다른 마스크 원하면 고객센터 연결

### 조회 데이터 및 필드
| 문의 유형 | Object | 주요 필드 |
|-----------|--------|-----------|
| 처방전 만료일 | Prescription__c | `End_Date__c`, `Manual_expiredate__c`, `Prescription_Status__c` |
| 순응 기간/사용시간 | Prescription__c | `Compilance_startdate__c`, `Compilance_enddate__c`, `Average_usage_time__c`, `usage_days__c` |
| 마스크 보험 구매 가능 여부 | Order / OrderItem | 해당 연도 마스크 구매 이력 (연 1회 제한) |
| 사용 중인 기기/마스크 | Asset | 기기 모델, 브랜드, 시리얼 |
| 계약 유형 | Contract__c | 보험임대 여부, 본인부담금 기준 |

### 본인인증 필드 (Account)
| 필드 | 비고 |
|------|------|
| `Name` | 고객 이름 |
| `Birthdate__c` (또는 `PersonBirthdate`) | 생년월일 — Org 확인 필요 |
| `MobilePhone` | SMS OTP 발송 대상 |

### Salesforce 구성 요소
| 구성요소 | 필요 여부 |
|----------|-----------|
| Knowledge + Data Cloud | ✅ 필수 |
| Account (생년월일 + 휴대폰 필드) | ✅ 필수 |
| Agentforce Action: Verify Customer Identity | ✅ 필수 (SMS OTP) |
| Prescription__c | ✅ 필수 |
| Asset | ✅ 필수 |
| Order / OrderItem | ✅ 필수 |
| Contract__c | ✅ 필수 |

## 시나리오 3: 상담사 이관 + Case 생성 (시나리오 1/2 미해결)

### 콘셉트
시나리오 1(FAQ) 또는 시나리오 2(개인정보 조회)에서 Agent가 해결하지 못한 경우,
고객 데이터와 대화 이력을 담은 Case를 자동 생성하고 상담사에게 이관.
상담사는 히스토리를 파악한 상태에서 고객과 대화하여 케이스 종료.

### 이관 트리거

**시나리오 1에서 미해결:**
- 기기 소음/이상 → 자가진단 안내 후에도 해결 안 됨 → AS 접수 필요
- Knowledge에 없는 내용 문의

**시나리오 2에서 미해결:**
- 본인인증 3회 실패
- 마스크 보험 구매 시 다른 마스크 선택 희망
- 처방전 갱신 / 계약 변경 등 담당자 처리 필요

**공통:**
- 고객이 직접 "상담사 연결" 요청

### Agent 흐름
```
시나리오 1 또는 2 진행 중
    ↓ 미해결 감지
Agent → "담당 상담사에게 연결해드리겠습니다" 안내
    ↓
Case 자동 생성
  - Account 연결 (시나리오 2 인증 완료 시 자동)
  - Subject: 문의 유형 자동 입력
  - 대화 이력 첨부 (BotSession → Case)
    ↓
Omni-Channel → 상담사 자동 배정
    ↓
상담사: Service Console에서 Case + 대화 히스토리 확인 후 응대
    ↓
상담사 ↔ 고객 대화 (LiveChatTranscript)
    ↓
Case 종료
```

### 상담사 화면 (Service Console)에서 보이는 것
- Case 제목 / 문의 유형
- 연결된 Account (고객명, 연락처, 계약 정보)
- Agent와 나눈 전체 대화 이력
- 관련 Prescription / Asset / Order 정보

### 대화 이력 저장 구조
```
Agentforce 구간  → BotSession + ConversationEntry
    ↓ 이관
상담사 구간      → LiveChatTranscript
    ↓
Case             ← 봇 이력 + 상담사 이력 + Account 통합 연결
```

### Salesforce 구성 요소
| 구성요소 | 필요 여부 |
|----------|-----------|
| Agentforce Escalation Action | ✅ 필수 |
| Case (자동 생성) | ✅ 필수 |
| Omni-Channel (상담사 라우팅) | ✅ 필수 |
| Service Console | ✅ 필수 |
| LiveChatTranscript | ✅ 자동 저장 |
| Account 연결 | ✅ 필수 |

> 상세 구성은 팀 미팅 후 확정
